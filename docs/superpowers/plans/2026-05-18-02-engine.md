# Plan 2: Scheduling Engine

## Overview

Plan 2 implements the pure-TypeScript scheduling engine — the heart of Plum Planner. It has no database calls; Server Actions invoke it, persist the result, then broadcast via Supabase Realtime.

The engine manages two task types: Fixed tasks (anchored to a specific date, never moved by the engine) and Fluid tasks (a prioritized stack that flows around Fixed tasks automatically). Key operations are `insertTask`, `deleteTask`, and `adjustTask` in `lib/engine/scheduler.ts`, supported by `compress` (gap-filling) and `validateConstraints` (soft violation detection).

## Key Invariants

1. Fluid tasks can never overlap Fixed tasks
2. Fixed tasks are never moved by the engine — only by explicit user action
3. Inserting onto an in-progress fluid task splits it (segments share `task_group_id`)
4. A task with `no_split` constraint falls back to push (not split) on interruption
5. `position` is only set on fluid tasks; Fixed tasks have `position = null`
